Here is a detailed breakdown of the State Machine Diagram for "Feature-5: Instant Previews."

### State Transitions Breakdown

Here is a table extracting the exact OCR text, mapping the states, triggers, and the resulting actions/transitions.

| Current State | Trigger / Guard Condition | Next State / Action |
| --- | --- | --- |
| **Idle** | `onSeriesCardHover(seriesId)` | HoverSession (starts at **Hovering**) |
| **Hovering** | `[hover >= 2s]` | PreviewFlow (starts at **TrailerIdLookup**) |
| **Hovering** | `onHoverEnd() [hover < 2s]` | **Cancelled** |
| **TrailerIdLookup** | `trailerId` | **TrailerLoadRequested** |
| **TrailerLoadRequested** | `loadTrailer(trailerId)` | **CacheDecision** |
| **CacheDecision** | `[cached == true]` | **PreviewPlaying** (starts at **MutedLoop**) |
| **CacheDecision** | `[cached == false]` | **NetworkFetch** (starts at **RequestCDN**) |
| **RequestCDN** | `CDNHit / trailerChunk(s)` | **PreviewPlaying** (starts at **MutedLoop**) |
| **RequestCDN** | `CDNmiss` | **FetchFromServer** |
| **FetchFromServer** | `trailerChunk(s)` | **PreviewPlaying** (starts at **MutedLoop**) |
| **PreviewPlaying** | `onPlayClicked()` | **TransitionToEpisode** |
| **TransitionToEpisode** | `transitionToEpisodePlayer()` | **EpisodeLoading** |
| **EpisodeLoading** | `getEpisodeData(episodeId)` | **EpisodeDataFetch** |
| **EpisodeDataFetch** | `startFullPlayback()` | Ends `PreviewFlow`, signals to exit |
| **PreviewFlow** | `onHoverEnd()` | **Cancelled** |
| **HoverSession** | `playSelected` | **FullPlayback** (starts at **Playing**) |
| **HoverSession** | `done/cancelled` | **Idle** |
| **Playing** | `exit` | End of Flow |

### Step-by-Step Flow Explanation

**1. The Initial Interaction (Idle to Hovering)**
The system sits in an `Idle` state until a user hovers over a series card, triggering `onSeriesCardHover(seriesId)`. This enters the main `HoverSession`. The system now waits in the `Hovering` state.

* **Cancel Condition:** If the user moves their mouse away before 2 seconds (`onHoverEnd() [hover < 2s]`), the flow immediately jumps to `Cancelled` and returns to `Idle`.

**2. Preparing the Preview (PreviewFlow)**
If the user keeps their mouse on the card for 2 seconds (`[hover>=2s]`), the system determines they are interested and enters `PreviewFlow`.

* It looks up the trailer ID (`TrailerIdLookup`).
* It requests the trailer to be loaded (`TrailerLoadRequested`).

**3. Resource Management (Cache vs. Network)**
To save bandwidth and load faster, the system evaluates a `CacheDecision`:

* If the trailer is already downloaded (`[cached == true]`), it skips downloading and goes straight to `PreviewPlaying`.
* If it is not downloaded (`[cached == false]`), it enters the `NetworkFetch` state. It first tries to request it from the closest Content Delivery Network (`RequestCDN`).
* If the CDN has it (`CDNHit`), it sends chunks of the video to `PreviewPlaying`.
* If the CDN doesn't have it (`CDNmiss`), it falls back to requesting it directly from the main server (`FetchFromServer`), which then sends the video chunks.



**4. Playing the Preview**
Once the video data is secured, the system enters `PreviewPlaying`, specifically looping the trailer without sound (`MutedLoop` via `playMutedLoop()`).

* **Cancel Condition:** At any point during this preview playback, if the user moves their mouse away (`onHoverEnd()`), the process halts, goes to `Cancelled`, and returns to `Idle`.

**5. Transitioning to Full Playback**
If the user decides to watch the show and clicks on the preview (`onPlayClicked()`), a sequence begins to swap the muted trailer for the actual episode:

* `TransitionToEpisode` readies the player.
* `EpisodeLoading` fetches the specific episode's metadata using `getEpisodeData(episodeId)`.
* `EpisodeDataFetch` completes the data load.
* `startFullPlayback()` is called, successfully completing the `HoverSession`.
* This triggers the `playSelected` transition, moving the system into the final `FullPlayback` state, where the user is actively `Playing` the episode until they `exit`.

### Example Scenario

Imagine you are browsing a streaming app and spot a sci-fi series.

1. You move your cursor over the poster. The system enters `Hovering`.
2. You decide it looks boring and move your mouse away after 1 second. The system jumps to `Cancelled` and does nothing.
3. Later, you hover over a comedy series. You pause there. After 2 seconds, the app decides to show you a preview.
4. It checks your app's temporary files (`CacheDecision`). You haven't seen this preview before, so it asks the local network node (`RequestCDN`) for the video.
5. The video arrives and starts playing silently on your screen (`MutedLoop`).
6. You like what you see and click the video. The app instantly stops the silent loop, loads Episode 1 (`EpisodeDataFetch`), opens the full-screen player, and the show begins (`FullPlayback`).
